# Claude Code 行動規範

このドキュメントは、Claude Code がこのプロジェクトで開発を行う際のグランドルールを定義する。

## 1. 開発ワークフロー

**仕様ファースト・テストファースト**を遵守する。コーディングを開始する前に、必ず以下の順序でコマンドを実行すること：

1. `/create-doc` - 会話内容から仕様書を作成
2. `/create-test-cases` - 仕様からテストケースを導出
3. `/implement-code` - コード実装

この順序を飛ばしてコーディングを開始してはならない。
実装方針は各コマンドに従う。

## 2. 自律的開発の範囲

### 2-1. 事前確認なしで進めてよいこと

- 既存の設計パターンに従った実装
- テスト通過を目的としたコード修正
- コンパイルエラーの解消
- 診断（error, suggestion）の解消
- コードフォーマット

### 2-2. 事前確認が必要なこと

- 新規ファイル・クラスの作成
- アーキテクチャや設計方針の変更
- 依存関係の追加
- 仕様書に記載のない機能の追加

### 2-3. 禁止事項

- 仕様書なしでのコーディング開始
- テストケースなしでの実装
- テストを無効化して実装を進めること

## 3. コミュニケーション

- 日本語でコミュニケーションを行う
- コードコメントは英語で書く
- コミットメッセージは Conventional Commits 形式を使用し、端的さを重視する
