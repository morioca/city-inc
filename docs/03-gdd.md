# City Inc（仮） - ゲームデザインドキュメント（GDD）

---

## 第1章：設計思想

### 首長視点の一貫性

プレイヤーは窓口職員ではなく、意思決定者である。個別の住民対応ではなく政策レベルの判断を行い、現場の細部ではなく数値とトレンドで状況を把握する。

### 段階的な深化

町長から市長、市長から県知事へと、段階的に規模と抽象度を上げながら、プレイの難易度と達成感を高めていく。

### 多様な価値観への対応

リベラルから保守まで、テック推進派から伝統重視派まで、多様な政治観を持つプレイヤーがそれぞれの「正解」を追求できる設計とする。正解は一つではない。

### コントロール可能な不確実性

イベントにより完全な予測は不可能だが、備えることで被害を軽減できる。「学べば有利になる」感覚を維持し、運任せではなく思考が報われる体験を提供する。

### 行政リアリティの設計

行政の「遅さ」をゲームから消すのではなく、遅い理由を可視化することで納得感と達成感を生む。年度予算（遅いが安定）と裁量枠・実験枠（速いがリスクあり）の二層構造により、戦略は年単位・戦術は月単位という行政らしいリズムを再現する。手続きを省略したり急いだりすることには、後からの反動やリスクを伴わせる。

### 支持率の戦略的役割

支持率は単なる評価指標ではなく、変革のスピードを左右する「政治的通貨」として機能する。強い支持があれば、通常は困難な政策も押し通せる。ルールを無視するのではなく、ルールを動かす力が増す設計とする。

### 変革の達成感

一定の条件（支持率、実績、任期など）を超えると、以後の意思決定が恒常的に速くなる仕組みを設ける。「制約を突破して自由度が増す」体験は、現実社会への不満を抱えるターゲット層に強く響く。

## 第2章：コアループ

### 基本サイクル

```
観察 → 判断 → 実行 → 結果確認 → 観察...
```

**観察（Observe）**
- 現在の支持率、財政状況、各指標を確認
- 住民からの陳情・要望を確認
- 競合候補（野党）の動向を確認

**判断（Decide）**
- 限られた予算をどの政策に配分するか決定
- 優先順位の設定
- トレードオフの選択

**実行（Act）**
- 選択した政策を実行
- 「次の月へ」ボタンで時間を進める

**結果確認（Review）**
- 政策の効果が数値で表示
- 住民の反応（支持率の変動）
- 予期せぬイベントの発生

## 第3章：ゲームシステム

### 3-1. ターン制設計

月次ターン制を採用する。

**採用理由：**
- 政策の効果は即座に現れず、数ヶ月〜数年のスパンで評価される
- 月次にすることで「今月何をするか」という判断の粒度が適切
- 隙間時間でのプレイに最適化

**戦略と戦術の分離：**
- 現実の行政は年単位（会計年度）で方針と予算配分を決定する
- 月ごとの状況評価と運用判断は現実的だが、月ごとに自由な予算組み替えや新規政策の即時実装は非現実的
- 本ゲームでは「戦略＝年」「戦術＝月」という二層構造で行政らしいリズムを再現する

### 3-2. 予算配分システム

**収入源：**
- 税収（住民税、法人税、固定資産税）
- 国からの交付金
- 使用料・手数料
- 公債発行（借金）

**支出カテゴリ（6分野）：**
1. 福祉・医療
2. 教育・子育て
3. 産業振興
4. インフラ整備
5. 防災・安全
6. 観光・文化

**設計ポイント：**
- 収入は自動計算（政策で間接的に影響）
- 支出は手動配分（プレイヤーの意思決定）
- 収支バランスが崩れると財政危機イベント発生

**予算の二層構造：**
- 本予算（年度予算）： 年度開始時に決定。変更には議会承認や補正予算が必要で時間がかかる
- 裁量枠・実験枠： 首長の判断で機動的に使える予算。規模は小さいが即座に実行可能
- 年度途中でも予備費や運用調整で一定の変更は可能だが、大きな方針転換には手続きと時間を要する設計とする

### 3-3. 政策システム

**政策の構造：**
- 各カテゴリに複数の政策オプション
- 政策にはコスト、効果期間、効果範囲がある
- 一度に実行できる政策数に上限

**政策の例（観光・外国人対策カテゴリ）：**

| 政策名 | コスト | 効果 | リスク |
|--------|--------|------|--------|
| 観光客数制限 | 低 | オーバーツーリズム緩和 | 観光収入減 |
| 多言語標識設置 | 中 | 外国人満足度向上 | 伝統派の反発 |
| 観光税導入 | 低 | 財源確保 | 観光業界の反発 |

### 3-4. 支持率システム

**支持率の構成：**
- 全体支持率 = 各層の支持率の加重平均
- 層ごとに重視する政策が異なる

**住民層：**

| 層 | 重視する政策 |
|----|--------------|
| 高齢者層 | 福祉・医療 |
| 子育て世代 | 教育・子育て |
| 事業者層 | 産業振興・規制緩和 |
| 若年層 | 雇用・住居費 |

**支持率の影響：**
- 支持率が低下すると議会で政策が通りにくくなる
- 選挙で落選するとゲームオーバー

**支持率＝スピードを上げる通貨：**
- 高い支持率は政治的正統性となり、通常より速い政策実行や例外的な予算措置を可能にする
- 支持率を消費して強引な政策を通すことも可能（ただし支持率は減少）
- 一定の支持率と実績を積み重ねると、恒常的に意思決定が速くなる「変革達成」状態に到達できる

### 3-5. イベントシステム

**定期イベント：**
- 選挙（4年ごと）
- 予算編成（毎年度開始時）
- 議会対応（四半期ごと）

**ランダムイベント：**
- 自然災害（地震、台風、大雪）
- 経済変動（景気後退、企業誘致チャンス）
- 社会問題（感染症流行、人口流出）
- スキャンダル（職員の不祥事）

**設計思想：**
- 「コントロール可能な不確実性」を提供
- 完全な予測は不可能だが、備えることで被害軽減可能
- 「学べば有利になる」感覚を維持

### 3-6. 委譲システム

**委譲の仕組み：**
- 各部局に部局長を配置
- 部局長に政策決定を委譲可能
- 委譲すると自動で政策が実行される

**委譲の必要性：**
- 規模が大きくなると全てを手動管理は不可能
- 「信頼して任せる」か「自分でコントロールする」かの選択

## 第4章：情報設計

### 4-1. 階層的情報設計（3層構造）

**第1層：俯瞰ビュー（マップ画面）**
- 地域全体の概況（支持率、財政状況、人口動態）
- 各エリアの状態をアイコンで表示

**第2層：部門/エリア詳細画面**
- 個別エリアの詳細データ
- 政策カテゴリ（福祉、産業、インフラ等）の状況

**第3層：分析・レポート画面**
- 詳細な統計データ
- 時系列グラフ
- 住民の声（テキストフィードバック）

### 4-2. 段階的複雑性解放

チュートリアルは設けず、段階的な機能解放でプレイしながら自然に覚える設計とする。

**Phase 1：町長として基礎を学ぶ**
- 限定された予算と政策オプション
- 基本的な住民ニーズへの対応
- 財政収支の基礎

**Phase 2：市長として組織経営**
- 複数の部局管理
- 議会との交渉
- 中期計画の策定

**Phase 3：県知事として広域行政**
- 複数市町村の調整
- 国との折衝
- 大規模プロジェクト

## 第5章：UI/UX設計

### 5-1. デザイン原則

**縦持ち（Portrait）を採用**
- 片手操作が可能で、通勤中など隙間時間にプレイしやすい
- ターン制のため「考える→タップで進める」のサイクルと相性が良い
- SNS等の他アプリも縦持ちが主流で、アプリ切り替えがスムーズ

**引き算のデザイン**
- 装飾的要素を排除
- 数値とその変化に集中
- ビジネスアプリケーションのような落ち着いた配色

**「首長の机」というメタファー**
- 画面は首長の執務室をイメージ
- 書類（レポート）を見る感覚で情報を確認
- 決裁印を押すように政策を承認

### 5-2. カラースキーム

| 用途 | 色 | 意図 |
|------|-----|------|
| メインカラー | ダークネイビー | 信頼感、公的機関らしさ |
| アクセントカラー | ゴールド | 首長の威厳 |
| 背景 | オフホワイト | 書類の紙色 |
| 警告 | オレンジ | 注意喚起 |
| 危機 | レッド | 緊急事態 |

### 5-3. 数値の表現

**前期比較を常に表示**
- 「人口 125,000人 (+0.5%)」
- 「支持率 62% (-3pt)」

**色による状態表示**
- 上昇：グリーン
- 横ばい：グレー
- 下降：レッド

**トレンドグラフの活用**
- 重要指標は折れ線グラフで推移を表示
- 「点」ではなく「線」で判断できるように

### 5-4. 画面構成（6画面）

**1. メイン画面（マップビュー）**
- 上部：支持率、財政残高、現在日付
- 中央：地域マップ（エリアごとに色分け）
- 下部：「次の月へ」ボタン、メニューアイコン
- エリア色分け：良好（緑）、注意（黄）、危機（赤）

**2. 政策画面**
- カテゴリタブ（福祉、教育、産業...）
- 実行可能な政策リスト
- 各政策の詳細（コスト、予想効果、影響層、期間）

**3. 予算画面**
- 収入の内訳（円グラフ）
- 支出の内訳（円グラフ）
- 収支バランス表示
- 各カテゴリへの配分スライダー
- 公債残高（借金）の推移

**4. 住民の声画面**
- 住民からの陳情リスト
- 各陳情の詳細と関連政策
- SNS風のフィードバック表示

**5. 統計・分析画面**
- 主要KPIダッシュボード
- 時系列グラフ（選択した指標）
- 表示可能指標：人口動態、経済指標、生活指標

**6. 設定画面**
- ゲーム設定
- セーブ/ロード

## 第7章：目標構造

### 短期目標（毎月）

- 今月の予算を黒字で終える
- 支持率を維持する
- 緊急課題に対応する

### 中期目標（1〜4年）

- 任期中の公約を達成する
- 特定の指標を目標値に到達させる
- 次の選挙で再選する

### 長期目標（複数任期）

- 町から市、市から県へ昇格
- 全国トップクラスの指標を達成
- 特定の称号（「福祉の聖地」「産業都市」等）を獲得
